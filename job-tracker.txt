#1 Step

Create a premium SaaS design system called "KodNest Premium Build System".

This is not a student project. This must feel like a serious B2C product company.

DESIGN PHILOSOPHY:
- Calm, Intentional, Coherent, Confident
- Not flashy, not loud, not playful, not hackathon-style
- No gradients, no glassmorphism, no neon colors, no animation noise

COLOR SYSTEM:
- Background: #F7F6F3 (off-white)
- Primary text: #111111
- Accent: #8B0000 (deep red)
- Success: muted green
- Warning: muted amber
- Use maximum 4 colors across entire system

TYPOGRAPHY:
- Headings: Serif font, large, confident, generous spacing
- Body: Clean sans-serif, 16–18px, line-height 1.6–1.8, max 720px for text blocks
- No decorative fonts, no random sizes

SPACING SYSTEM (consistent scale):
8px, 16px, 24px, 40px, 64px
Never use random spacing like 13px, 27px, etc. Whitespace is part of design.

GLOBAL LAYOUT STRUCTURE:
Every page must follow: [Top Bar] → [Context Header] → [Primary Workspace + Secondary Panel] → [Proof Footer]

TOP BAR:
- Left: Project name
- Center: Progress indicator (Step X / Y)
- Right: Status badge (Not Started / In Progress / Shipped)

CONTEXT HEADER:
- Large serif headline, 1-line subtext, clear purpose, no hype language

PRIMARY WORKSPACE (70% width):
- Where the main product interaction happens
- Clean cards, predictable components, no crowding

SECONDARY PANEL (30% width):
- Step explanation (short)
- Copyable prompt box
- Buttons: Copy, Build in Lovable, It Worked, Error, Add Screenshot
- Calm styling

PROOF FOOTER (persistent bottom section):
Checklist style: □ UI Built □ Logic Working □ Test Passed □ Deployed
Each checkbox requires user proof input.

COMPONENT RULES:
- Primary button = solid deep red, Secondary = outlined
- Same hover effect and border radius everywhere
- Inputs: clean borders, no heavy shadows, clear focus state
- Cards: subtle border, no drop shadows, balanced padding

INTERACTION RULES:
- Transitions: 150–200ms, ease-in-out, no bounce, no parallax

ERROR & EMPTY STATES:
- Errors: explain what went wrong + how to fix, never blame user
- Empty states: provide next action, never feel dead

Everything must feel like one mind designed it. No visual drift. Return confirmation once design system is created. Do NOT add product features yet.

#5 Step

Upgrade the existing Job Notification Tracker by implementing real preference logic and deterministic match scoring.

NON-NEGOTIABLE:
- Do NOT change routes.
- Do NOT alter design system.
- Do NOT remove existing features.

------------------------------------------
1) IMPLEMENT PREFERENCES LOGIC
------------------------------------------

On /settings:

Fields:
- roleKeywords (comma-separated text input)
- preferredLocations (multi-select dropdown)
- preferredMode (checkboxes: Remote, Hybrid, Onsite)
- experienceLevel (dropdown)
- skills (comma-separated text input)
- minMatchScore (slider 0–100, default 40)

Save preferences to localStorage as:
jobTrackerPreferences

On load:
If preferences exist, prefill form.

------------------------------------------
2) IMPLEMENT MATCH SCORE ENGINE
------------------------------------------

For each job on /dashboard, compute matchScore using these exact rules:

+25 if any roleKeyword appears in job.title (case-insensitive)
+15 if any roleKeyword appears in job.description
+15 if job.location matches preferredLocations
+10 if job.mode matches preferredMode
+10 if job.experience matches experienceLevel
+15 if overlap between job.skills and user.skills (any match)
+5 if postedDaysAgo <= 2
+5 if source is LinkedIn

Cap score at 100.

Display matchScore as a badge on job card:
- 80–100: green
- 60–79: amber
- 40–59: neutral
- <40: subtle grey

------------------------------------------
3) ADD "SHOW ONLY MATCHES" TOGGLE
------------------------------------------

On /dashboard:

Add toggle:
"Show only jobs above my threshold"

When enabled:
Filter jobs where matchScore >= minMatchScore

------------------------------------------
4) UPDATE FILTER BAR LOGIC
------------------------------------------

Filters must combine logically (AND behavior):

- Keyword search
- Location filter
- Mode filter
- Experience filter
- Source filter

Sorting must work:
- Latest (postedDaysAgo ascending)
- Match Score (descending)
- Salary (simple numeric extract sort if possible)

------------------------------------------
5) EDGE CASE HANDLING
------------------------------------------

If preferences not set:
Show banner:
"Set your preferences to activate intelligent matching."

If no matches found:
Show premium empty state:
"No roles match your criteria. Adjust filters or lower threshold."

------------------------------------------
6) PERFORMANCE RULE
------------------------------------------

Do not re-render entire list unnecessarily.
Keep UI smooth.
No console errors.

------------------------------------------
OUTPUT REQUIRED
------------------------------------------

After implementation:
Return:

1) Confirmation that scoring rules exactly match specification.
2) Brief explanation of matchScore calculation logic.
3) Verification steps:
   - Example preference to test
   - Expected match behavior
